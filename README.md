# Как запустить сайт на GitHub Pages — очень простая инструкция

Этот файл написан **для обычного человека**, без лишней технички.
Если ты просто хочешь:
- залить сайт на GitHub,
- открыть его по ссылке,
- редактировать текст/фото через админку,
- видеть общую статистику заходов,

то просто иди **по шагам сверху вниз**.

---

# Что у тебя уже есть

В папке проекта лежат такие файлы:
- `index.html` — сама страница сайта
- `styles.css` — внешний вид сайта
- `main.js` — логика сайта, админки, пароля и статистики
- `README.md` — эта инструкция
- `SUPABASE_SETUP.md` — отдельная инструкция для базы данных

---

# Что ты получишь в итоге

После настройки у тебя будет:
- сайт на GitHub Pages,
- ссылка вида `https://твой-ник.github.io/название-репозитория/`,
- админка прямо на сайте,
- пароль на админку,
- загрузка фото,
- общая статистика заходов и кликов,
- возможность редактировать сайт без переписывания кода.

---

# Самый короткий план

Если в двух словах, то нужно сделать вот это:

## Часть 1
1. Создать аккаунт на GitHub
2. Создать репозиторий
3. Загрузить туда файлы сайта
4. Включить GitHub Pages

## Часть 2
1. Создать проект в Supabase
2. Вставить SQL
3. Вписать `URL` и `anon key` в админку сайта
4. Нажать `cloud push`

После этого сайт будет работать нормально и данные будут общими для всех.

# ЧАСТЬ 1 — как залить сайт на GitHub Pages

## Шаг 1. Зарегистрируйся на GitHub

Если у тебя ещё нет аккаунта:
1. Открой сайт: `https://github.com`
2. Нажми кнопку **Sign up**
3. Создай аккаунт
4. Подтверди почту

Если аккаунт уже есть — просто войди в него.

---

## Шаг 2. Создай новый репозиторий

Репозиторий — это просто папка проекта на GitHub.

Что нажимать:
1. В правом верхнем углу GitHub нажми **+**
2. Выбери **New repository**
3. В поле **Repository name** напиши, например:
   - `links`
   - или `my-site`
   - или любое своё название
4. Ниже выбери **Public**
5. Нажми **Create repository**

Готово. Репозиторий создан.

---

## Шаг 3. Загрузи файлы сайта в репозиторий

После создания репозитория GitHub покажет страницу.

Если репозиторий пустой, сделай так:
1. Нажми **uploading an existing file**
2. Открой свою папку проекта
3. Перетащи на страницу GitHub вот эти файлы:
   - `index.html`
   - `styles.css`
   - `main.js`
   - `README.md`
   - `SUPABASE_SETUP.md`
4. Пролистай вниз
5. Нажми зелёную кнопку **Commit changes**

Теперь файлы загружены.

---

## Шаг 4. Включи GitHub Pages

Теперь нужно сделать так, чтобы GitHub начал показывать сайт как веб-страницу.

Что нажимать:
1. Открой свой репозиторий
2. Нажми вкладку **Settings**
3. Слева найди пункт **Pages** и нажми на него
4. В блоке **Build and deployment** найди поле **Source**
5. Выбери **Deploy from a branch**
6. Ниже в поле **Branch** выбери:
   - ветка: `main`
   - папка: `/ (root)`
7. Нажми **Save**

Теперь GitHub начнёт публиковать сайт.

---

## Шаг 5. Получи ссылку на сайт

После включения GitHub Pages:
1. Подожди 1–3 минуты
2. Обнови страницу **Settings → Pages**
3. Там появится ссылка на сайт

Обычно ссылка выглядит так:
- `https://ТВОЙ-НИК.github.io/НАЗВАНИЕ-РЕПОЗИТОРИЯ/`

Пример:
- `https://iopio.github.io/links/`

Открой её — сайт уже будет работать.

# ЧАСТЬ 2 — как сделать, чтобы админка и статистика были общими для всех

## Важно понять простыми словами

Сейчас сайт статический.
Это значит:
- GitHub Pages просто показывает страницу,
- но сам GitHub Pages **не хранит базу данных**,
- поэтому для общей админки и общей статистики нужен отдельный сервис.

Мы уже сделали поддержку такого сервиса.
Он называется **Supabase**.

Если **не подключать Supabase**, то:
- сайт откроется,
- админка будет работать,
- но изменения будут сохраняться только у тебя в браузере.

Если **подключить Supabase**, то:
- изменения будут общими,
- статистика будет общей,
- сайт станет работать как надо на GitHub Pages.

---

## Шаг 6. Зарегистрируйся в Supabase

1. Открой: `https://supabase.com`
2. Нажми **Start your project**
3. Зарегистрируйся
4. Создай новый проект

Когда проект создастся, ты попадёшь в панель управления Supabase.

---

## Шаг 7. Найди `Project URL` и `anon key`

Что нажимать:
1. В Supabase открой свой проект
2. Слева нажми **Project Settings**
3. Потом нажми **API**
4. Найди там:
   - **Project URL**
   - **Project API keys**
   - **anon public** key

Тебе нужно скопировать:
- `Project URL`
- `anon public key`

Пока просто сохрани их куда-нибудь во временный текстовый файл.

---

## Шаг 8. Создай таблицы в Supabase

Теперь нужно один раз вставить SQL.

Что нажимать:
1. В Supabase слева нажми **SQL Editor**
2. Нажми **New query**
3. Открой файл `SUPABASE_SETUP.md` у себя в проекте
4. Скопируй оттуда весь SQL-код
5. Вставь его в окно SQL Editor
6. Нажми **Run**

После этого создадутся таблицы для:
- контента сайта,
- статистики сайта.

---

## Шаг 9. Открой свой сайт и зайди в админку

Теперь вернись на свой сайт по GitHub Pages ссылке.

Что делать:
1. Открой сайт
2. Найди блок `# admin panel`
3. Нажми кнопку **open admin**

Если заходишь первый раз:
1. В поле пароля введи **свой новый пароль**
2. Нажми **unlock**

Это создаст пароль для админки.

Если пароль уже был создан раньше:
1. Просто введи его
2. Нажми **unlock**

---

## Шаг 10. Включи cloud mode

После разблокировки админки ты увидишь поля для облака.

Что туда вставлять:

### Поле `Cloud mode (Supabase)`
- включи галочку

### Поле `Supabase URL`
- вставь сюда `Project URL` из Supabase

### Поле `Supabase anon key`
- вставь сюда `anon public key` из Supabase

### Поле `Content table`
- оставь так: `site_content`

### Поле `Events table`
- оставь так: `site_events`

### Поле `Content row key`
- оставь так: `main`

---

## Шаг 11. Сохрани сайт в облако

Теперь нужно первый раз отправить контент сайта в Supabase.

Что нажимать:
1. Нажми **save changes**
2. Потом нажми **cloud push**

Это загрузит текущий контент сайта в облако.

---

## Шаг 12. Подтяни статистику

Теперь нажми:
- **cloud sync stats**

После этого облачная статистика тоже подключится.

# Как теперь пользоваться сайтом

---

## Как поменять текст на сайте

1. Открой сайт
2. Нажми **open admin**
3. Введи пароль
4. Нажми **unlock**
5. Найди нужные поля:
   - `Welcome title`
   - `Welcome description`
   - `About text`
   - `Skills`
   - `Projects`
   - `Contacts`
   - `Donations`
6. Измени текст
7. Нажми **save changes**

Если cloud mode включён правильно, изменения станут общими.

---

## Как поставить своё фото

Есть два варианта.

### Вариант 1 — загрузить файл
1. Открой админку
2. Разблокируй её паролем
3. Найди поле `Profile photo upload`
4. Выбери свой файл
5. Нажми **save changes**

### Вариант 2 — вставить ссылку на фото
1. Открой админку
2. Найди поле `Profile photo URL`
3. Вставь прямую ссылку на картинку
4. Нажми **save changes**

---

## Как изменить подпись на большой фото-карточке

В админке есть поля:
- `Profile photo title` — большой текст внизу карточки
- `Profile photo badge` — значок рядом с текстом
- `Profile photo link` — ссылка, куда ведёт карточка

После изменения нажми **save changes**.

---

## Как сменить пароль админки

1. Открой админку
2. Разблокируй её
3. Нажми **change password**
4. Введи текущий пароль
5. Введи новый пароль
6. Подтверди

---

## Как заблокировать админку

Если закончил редактирование:
1. Нажми **lock admin**

Админка снова закроется и будет ждать пароль.

---

# Как понять, что всё настроено правильно

Проверь вот так:

## Проверка 1
Открой сайт — он должен нормально загрузиться.

## Проверка 2
Нажми `open admin` — должна открыться админка.

## Проверка 3
Введи пароль — админка должна разблокироваться.

## Проверка 4
Измени любой текст и нажми `save changes` — текст должен поменяться.

## Проверка 5
Обнови страницу — изменения должны остаться.

## Проверка 6
Открой сайт в другом браузере или с телефона:
- если всё настроено через Supabase правильно,
- ты увидишь те же самые изменения.

## Проверка 7
Нажми на внешнюю ссылку на сайте, потом вернись назад и нажми `cloud sync stats` — число кликов должно увеличиться.

# Если что-то не работает

## Проблема: сайт не открылся по GitHub Pages ссылке
Что делать:
1. Проверь, включён ли GitHub Pages
2. Проверь, выбрана ли ветка `main`
3. Проверь, выбрана ли папка `/ (root)`
4. Подожди 1–3 минуты
5. Обнови страницу

---

## Проблема: админка есть, но данные не общие
Значит Supabase не подключён или подключён неправильно.

Проверь:
1. Включена ли галочка `Cloud mode (Supabase)`
2. Правильно ли вставлен `Supabase URL`
3. Правильно ли вставлен `Supabase anon key`
4. Выполнен ли SQL из `SUPABASE_SETUP.md`
5. Нажимал ли ты `cloud push`

---

## Проблема: `cloud push` или `cloud pull` выдаёт ошибку
Обычно причина одна из этих:
- неправильный URL,
- неправильный ключ,
- SQL не был выполнен,
- таблицы не создались,
- политики доступа не создались.

Что делать:
1. Ещё раз открыть `SUPABASE_SETUP.md`
2. Скопировать SQL полностью
3. Снова вставить в **SQL Editor**
4. Нажать **Run**
5. Потом снова вернуться на сайт
6. Снова нажать `cloud push`

---

## Проблема: забыл пароль от админки
Пароль хранится в браузере.

Что можно сделать:
1. Открыть сайт в другом браузере
2. Задать новый пароль там

Либо:
1. Открыть инструменты разработчика браузера
2. Очистить `localStorage` для сайта
3. Открыть сайт заново
4. Создать новый пароль

---

# Очень важный момент

Пароль админки сейчас сделан **на стороне браузера**.
Это удобно, но это не супер-защита уровня банка.

То есть:
- для обычного личного сайта — нормально,
- для простого использования — нормально,
- для серьёзной закрытой админки — уже нужен отдельный backend с авторизацией.
